# Stage 7.05: hardening, regression, parity review

> Part of the stage-7 sub-plan ([specs.md](specs.md); index [../07-ingame-panel.md](../07-ingame-panel.md)). This closing pass covers degradation paths, size and edge sweeps, native-screen regression, and comparison with the Web flow. It follows the human-control plan's `08-comparability-review.md` precedent.

## Objective

The panel fails soft everywhere, both observer flavors satisfy the canonical capability matrix, the watch-items are exercised rather than assumed, nothing native regressed, and the in-game experience is point-by-point comparable to the Web flow it mirrors.

## Work items

1. **Degradation paths (civ5-mod).** `pcall` guards around every push-event payload read (a malformed table renders an error row, never a Lua error: mirroring the HumanPanel's defensive style); the ~10 s transport-ack and ~90 s reply timeouts surface their Retry affordances; the `hasEnvoy:false` and services-down cases show the "envoy unavailable" body; the oversized-deal refusal shows its reason.
2. **Failure-mode sweep (whole stack).** Kill vox-agents mid-conversation (pending send times out into Retry; no `Lua.log` errors; recovery on restart + reopen); restart bridge-service (push functions still reachable, or the re-registration fallback engages); force an `overflowed()` push and a `ThreadBusyError` (both surface as error statuses, panel stays usable).
3. **Watch-item sweep (per specs).** Open a deal draft, add several ordinary terms and promises, then run server-side `inspect-deal`. Confirm the visible draft and submitted payload remain unchanged because `draftItems` and `draftPromises` rebuild the shared scratch deal. Test delimiter stripping in both directions. During a long reply, confirm deltas arrive at least one second apart while the action FIFO remains busy, and reopen the panel to confirm `Begin.busy` arrives before the turn finishes. Keep the push queue below the bridge's auto-pause threshold. Page through several hundred transcript rows without frame hitches; add lazy rendering only if profiling shows a problem. Exercise a normal player, a human strategist, and a pure observer against every row of the [canonical capability matrix](specs.md#observer-flavors-and-the-effective-seat). Confirm pinned deal legality and values are computed from the pinned civ. For an out-of-range pure-observer slot, confirm deal controls are unavailable and a forged `AsObserver` deal action fails closed. Also forge a seated deal action from an observer slot without the matching effective-seat state and confirm it fails closed, so removing `AsObserver` cannot bypass the seat-holder check.
4. **Trade footer fail-safe (civ5-mod).** In Vox mode, force every TradeLogic button refresh with the wrapper listener installed and confirm it synchronously replaces the stock Propose and Cancel callbacks, configures the mounted actions, and re-enables only valid controls. Then remove the listener and refresh again. Both stock controls must remain disabled, so no native AI or network proposal path can run from the wrapper's effective seat when footer ownership is missing or stale.
5. **Regression pass.** Native trade screen, Discuss/Demand flows, and the notification panel behave stock (rename discipline, guarded click hook); the Web chat/deal routes behave exactly as before the stage-04 extraction (status codes included); the human-control diplo poke still works. In a normal game, confirm `AsObserver` remains absent, the `Network.SendChangeWar` path is untouched, and `post-notification` behaves exactly as before for normal seat IDs. Re-run `update_md5.py` and confirm the mod deploys clean.
6. **Comparability review (doc).** After implementation, append a short comparison for each Web capability: history, greeting, streaming with revealable detail, deal cards and statuses, propose/counter/accept/reject/retract, close, and the stale-proposal guard. Record where the panel matches the Web, deliberately simplifies it (status-only streaming and no draft replay), or defers work. Add one observer row group that records the intentional identity and durability difference: Web observer chats use the ephemeral `-1` sentinel, while an in-game pure observer uses the durable diplomacy-thread machinery keyed by the real observer slot ID. Feed the result into parent stages 8 and 9.

## Verify

Every listed sweep is executed against a live game with results noted inline (this stage's work items are its verify), including both observer flavors, the unsupported out-of-range pure-observer deal case, both forged-deal fail-closed cases, the missing-listener footer fail-safe, and the normal-game regression checks. The comparability table is filled in, including the observer divergence row group, with no open `Lua.log` errors across the full pass.

## Done when

All degradation and watch-item behaviors are observed working for normal and observer sessions, forged observer deal actions fail closed, an unclaimed Vox footer stays inert, native and Web surfaces are regression-free, and the comparability review documents exactly how the in-game panel relates to the Web flow.
