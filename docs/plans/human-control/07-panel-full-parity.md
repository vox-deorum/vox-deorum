# Stage 7 — In-game panel v3: the rest of the action space (full parity) ✅ DONE

> Part of the human-control plan. Shared design and watch-items live in [README.md](README.md); requirements in [specs.md](specs.md).

> **Status: implemented.** The panel now expresses the full Flavor-mode action space through the approved mockup's master-detail layout. What was built:
> - [`civ5-mod/UI/VoxDeorumHumanPanel.xml`](../../../civ5-mod/UI/VoxDeorumHumanPanel.xml) — the dialog grew to 980×700 and became master-detail: a **leader context row** (the human civ's leader portrait, trait name + description with full-text tooltip, and unique unit/building/improvement icons with "replaces …" tooltips — read from the game database via the `PopulateUniques.lua` queries, only the human's own civ), a **left nav** of the six categories (Grand Strategy, Flavors, Next Research, Next Policy, Persona, Relationships — each an instanced button with a one-line current-value summary and an `[ICON_BULLET]` badge when edits are staged), a **right pane** (one shared `ScrollPanel`+`Stack` rebuilt per category), and a footer with a **wrapping multi-line staged-changes summary label** (full list in its tooltip; per-change undo happens at the controls — sliders have Reset, lists unstage by re-picking the current option — instead of the mockup's chip row, which Civ's UI can't wrap), the rationale box (MaxLength 300), Keep Status Quo, and Submit. New instances: `OptionInstance` (generalized from stage 6's `ResearchInstance`), `TextInstance`, `GroupInstance` (collapsible header; members are stack siblings toggled hidden), `SliderInstance` (name, current→staged numbers, description, −/slider/+/Reset line, semantic tick labels), `RelationInstance` (met-leader portrait + last-set note), `NavInstance`, `UniqueInstance`. All stage 5/6 corner-widget behaviors are unchanged.
> - [`civ5-mod/UI/VoxDeorumHumanPanel.lua`](../../../civ5-mod/UI/VoxDeorumHumanPanel.lua) — staging mirrors the mockup: `m_staged = {GrandStrategy, Flavors{}, Technology, Policy, Persona{}, Relationships{}}`, where setting a control back to its current value unstages it. Single-select lists for grand strategy (alphabetical, no icons), research (stage 6's GridX ordering + tech icons), and policy (continuing-branch entries before new branches; icons matched via `GameInfo.Policies`/`PolicyBranchTypes` on the suffix-stripped display name; the current selection matched suffix-free because `Policy.Next` says "(Policy)" while option keys say "(Continuing X Branch)"). Sliders for flavors (0–100 step 5, grouped per `flavors.json`'s three blocks), persona (1–10 step 1, grouped per the `set-persona` schema sections), and per-met-civ Public/Private stances (−100..100 step 5); rows refresh in place so dragging never re-renders, and report keys missing from the hardcoded groups fall into a trailing "Other" group. Met civs are enumerated from the `Players` table (alive majors the human's team has met) and matched to report `Relationships` by `GetCivilizationShortDescription()` — the same key the server stores. **Submit fires ONE `HumanDecision`** carrying only changed fields (`Flavors`/`Persona` as delta maps, `Relationships` as an array with the unchanged dimension riding at its current value, since `set-relationship` defaults absent dimensions to 0) plus `DeliberationMs`, measured in Lua from first dialog open to submission; **Keep Status Quo uses a two-click confirm** when edits are staged (the button relabels to "Discard N changes?"; any other interaction disarms) instead of the mockup's browser `confirm()`. Every category renders a "no options" note when its report section is missing, so a degraded report still allows keep-status-quo. Display names come from `TXT_KEY_VD_HUMAN_NAME_<KEY>` overrides with a de-PascalCase fallback; persona descriptions are TXT keys copied verbatim from the `set-persona` schema (the report doesn't carry them, and adding them would change the LLM's context).
> - [`civ5-mod/Text/VoxDeorum_Text.xml`](../../../civ5-mod/Text/VoxDeorum_Text.xml) — category titles/intros, group titles, tick labels, nav summaries, staged-summary/confirm/accepted texts, relationship labels, 13 display-name overrides, and the 26 persona descriptions. Stage 6's research-specific `CURRENT_TAG`/`EARLIER_RATIONALE`/chip keys were generalized (`TXT_KEY_VD_HUMAN_CURRENT_TAG`, `TXT_KEY_VD_HUMAN_EARLIER_RATIONALE`, `TXT_KEY_VD_HUMAN_AUTOPLAY_CHIP_DECISION`, `TXT_KEY_VD_HUMAN_ACCEPTED_SUB_DECISION`).
> - [`civ5-dll/CvGameCoreDLL_Expansion2/CvConnectionService.cpp`](../../../civ5-dll/CvGameCoreDLL_Expansion2/CvConnectionService.cpp) — the `lua_call` args buffer was bumped 16 KB → 64 KB, closing stage 6's headroom watch-item (the whole `OptionsReport` travels through it; a late-game report plus ArduinoJson node overhead could have overflowed into the "no options" degradation). **Requires a DLL rebuild**; the mod-side changes alone do not.
> - [`civ5-mod/VoxDeorum.modinfo`](../../../civ5-mod/VoxDeorum.modinfo) — `update_md5.py` re-run (no new files).
> - [`vox-agents/src/strategist/agents/human-strategist.ts`](../../../vox-agents/src/strategist/agents/human-strategist.ts) — already mapped every submitted field onto its action tool since stage 3 (`set-flavors` carries `GrandStrategy`+`Flavors` in one call; `set-research`; `set-policy`; `set-persona`; per-civ `set-relationship`), replicating the single rationale; verified, only a stale "lands in a later stage" comment was freshened. Type-check and the decision-bus tests pass.

## Objective

Complete the panel to full LLM parity (spec §1): flavors, next research, policy (via the native screen if stage 6 adopted it), persona (the 26 values), and per-civ relationships — organized per the approved mockup so the core decision stays fast while persona and relationships remain accessible. The human runs in **Flavor mode**; the legacy **Strategy mode** (grand/economic/military strategies) is outdated and not supported for human decision-makers. After this stage the human's action space exactly equals the LLM's Flavor-mode action space.
